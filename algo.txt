2. Frother pump PI control
Use discrete-time sampled PI control for frother dosing.

Definitions:
- measured_count = latest bubble_count from vision metrics
- error = pi_setpoint - measured_count
- deadband = 10 counts
- running_duty = 0%
- max_auto_frother_duty = 50%
- full_scale_error = pi_setpoint

Frother behaviour:
- If measured_count is within pi_setpoint ± 10:
  - frother duty = 0%
  - reset no-response counter
  - bleed/decay integral
  - state = "ON_TARGET"

- If measured_count is above pi_setpoint + 10:
  - frother duty = 0%
  - bleed/decay integral faster
  - state should later be set by air logic, e.g. "AIR_NORMAL", "AIR_REDUCING", or "AIR_MINIMUM"

- If measured_count is below pi_setpoint - 10:
  - calculate proportional duty:
    error_ratio = error / full_scale_error
    clamp error_ratio from 0 to 1
    proportional_duty = pi_kp * max_auto_frother_duty * error_ratio

  - calculate integral bias:
    pi_integral += pi_ki * error * CONTROL_PERIOD
    clamp pi_integral from 0 to integral_limit
    integral_limit should default to 20.0 duty-percent contribution

  - raw_duty = proportional_duty + pi_integral
  - final frother duty = clamp raw_duty between 0 and 50
  - apply anti-windup:
    only accept integral growth when raw_duty is below max_auto_frother_duty
  - state = "FROTHER_PI_DOSING"

Recommended default tuning:
- pi_kp = 1.0
- pi_ki = 0.001
- pi_setpoint = 120
- max_auto_frother_duty = 50.0
- running_duty = 0.0
- deadband = 10.0

3. Saturation/no-response freeze protection
Implement saturation protection for the frother PI.

Purpose:
If the controller is dosing but the bubble count does not improve much after repeated sampled control cycles, freeze dosing to prevent reagent waste.

Rules:
- Only check saturation when count is still below the setpoint band.
- Compare current measured_count with the previous sampled control count.
- If frother pump was dosing at or above min_active_duty and count increased by less than response_threshold, increment no_response_cycles.
- response_threshold = 10 bubble counts.
- min_active_duty = 2%.
- If count improves by 10 or more, reset no_response_cycles to 0.
- If no_response_cycles reaches 5:
  - set dosing_frozen = True
  - frother duty = 0%
  - pi_integral = 0
  - auto_state = "FROZEN_NO_RESPONSE"
  - keep frozen until operator resets AUTO mode by switching manual then auto, or until set_pump_mode('auto') is called again.

4. Air pump automatic correction above setpoint
The air pump should only be automatically corrected when bubble count is clearly above the setpoint.

Air behaviour:
- Normal air duty = 65%
- Minimum air duty = 55%
- Air reduction starts only when count is above setpoint by 20 counts.
- Air reaches minimum duty when count is above setpoint by 50 counts.
- Between +20 and +50, reduce air linearly from 65% to 55%.

Formula:
overshoot = measured_count - pi_setpoint

If overshoot <= 20:
    air_duty = 65

If overshoot >= 50:
    air_duty = 55

If 20 < overshoot < 50:
    air_duty = 65 - ((overshoot - 20) / (50 - 20)) * (65 - 55)

Important:
- When count is below or inside the setpoint band, air should return to normal duty = 65%.
- Frother and air should not fight each other:
  - low count: frother PI acts, air = 65
  - on target: frother = 0, air = 65
  - slightly high: frother = 0, air = 65
  - clearly high: frother = 0, air reduces proportionally
  - very high: frother = 0, air = 55
